# Spec 01 — Angular Material and Theme

## Objective

Install and configure Angular Material with a custom theme (Material 3), using primary color `#003D7A` and secondary color `#78DE1F`, supporting a light/dark mode toggle.

## Installation

```bash
ng add @angular/material

```

During the schematic prompts:

- Choose **"Custom"** (or "None") instead of a prebuilt theme — we'll generate our own.
- Enable "Global Typography": yes
- Enable "Browser Animations": yes (`provideAnimationsAsync()` or `provideAnimations()`)

## Palette generation (Material 3)

Use the official schematic to generate the theme from the primary and secondary colors:

```bash
ng generate @angular/material:m3-theme --primary-color="#003D7A" --secondary-color="#78DE1F"

```

- If the installed schematic version doesn't accept these flags directly, run `ng generate @angular/material:m3-theme` without flags and provide the colors when prompted interactively (primary `#003D7A`, secondary `#78DE1F`).
- Leave tertiary/warn as auto-generated suggestions, unless you want to customize them later.
- This generates a SCSS file (e.g. `styles/_theme-colors.scss` or similar, depending on the version) with color tokens for both modes (light and dark), including `$primary-palette` and `$secondary-palette`.



## Expected structure in `styles.scss`

```scss
@use '@angular/material' as mat;
@use './theme-colors' as my-theme; // file generated by the schematic

html {
  @include mat.theme((
    color: (
      theme-type: light,
      primary: my-theme.$primary-palette,
      secondary: my-theme.$secondary-palette,
      tertiary: my-theme.$tertiary-palette,
    ),
    typography: Roboto,
    density: 0,
  ));
}

html.dark-theme {
  @include mat.theme((
    color: (
      theme-type: dark,
      primary: my-theme.$primary-palette,
      secondary: my-theme.$secondary-palette,
      tertiary: my-theme.$tertiary-palette,
    ),
  ));
}

```

> Adjust exact variable names based on the actual output of the `m3-theme` schematic (the installed Angular Material version may generate slightly different syntax).



## Using the secondary color

- The secondary color (`#78DE1F`) should be used sparingly, for visual accents — not as the dominant UI color (the overall look stays clean/neutral, with the primary `#003D7A` as the base).
- Good candidates for `color="secondary"` or the secondary palette:
  - Main action button on the Home page ("View Pokemon list")
  - Type chips on the detail page (spec 06), or a subset of them
  - Active/success state indicators, if needed



## ThemeService (light/dark toggle)

- Name: `ThemeService`
- Location: `src/app/core/services/theme.service.ts`
- Responsibilities:
  - Hold a `signal<'light' | 'dark'>` with the current mode.
  - `toggleTheme()` method: flips the value and adds/removes the `dark-theme` class on the `<html>` element (`document.documentElement.classList.toggle('dark-theme')`).
  - Persist the preference in `localStorage` (e.g. key `pokedex-theme`) and restore it on app startup (e.g. via `APP_INITIALIZER` or in the service constructor).
  - If there's no saved preference, respect the system's `prefers-color-scheme` as the initial value.



## UI toggle

- A button/icon (`mat-icon-button` with a sun/moon icon, using `MatIconModule`) in the `mat-toolbar` of the layout (see spec 02), calling `themeService.toggleTheme()`.



## Acceptance criteria

- The app renders with `#003D7A` as the primary color and `#78DE1F` available as the secondary color across Material components.
- The toggle button switches between light and dark instantly.
- Reloading the page keeps the last chosen mode (persisted).
